# robotics_playground

Learning ROS 2 project on Ubuntu 24.04 without physical hardware.

- **OS:** Ubuntu 24.04
- **ROS:** ROS 2 Jazzy
- **Language:** Python (ament_python)
- **Build system:** colcon

## Repository structure

```
robotics_playground/              # git repository root
├── .vscode/
│   ├── settings.json             # ROS 2 + VS Code integration
│   └── c_cpp_properties.json     # IntelliSense for C++
├── robotics_playground_ws/       # ROS 2 workspace
│   ├── src/
│   │   └── learning/             # educational package
│   ├── build/                    # generated by colcon (gitignored)
│   ├── install/                  # generated by colcon (gitignored)
│   └── log/                      # generated by colcon (gitignored)
└── README.md
```

## Setup

```bash
cd robotics_playground_ws
colcon build
source install/setup.bash
```

For Python packages, use `--symlink-install` during development — no rebuild needed after editing `.py` files, just restart the node:

```bash
colcon build --symlink-install
```

The VS Code integrated terminal sources `install/setup.bash` automatically on startup.

## ROS 2 CLI reference

### Topics

```bash
# List all active topics
ros2 topic list

# List with message types
ros2 topic list -t

# Show message type of a topic
ros2 topic type /learning/status

# Show publisher/subscriber count and type
ros2 topic info /learning/status

# Show structure of a message type
ros2 interface show std_msgs/msg/String

# Print incoming messages to terminal
ros2 topic echo /learning/status

# List all active nodes
ros2 node list
```

> Topics are only visible while at least one publisher or subscriber is running.

## Package: learning

Educational package for learning ROS 2 concepts step by step.

**Build type:** `ament_python`
**Dependencies:** `rclpy`, `std_msgs`

### Package structure

```
learning/
├── learning/
│   ├── constants.py          # shared topic/service names
│   └── topics/
│       ├── status_publisher.py
│       ├── status_subscriber.py
│       ├── command_publisher.py
│       └── command_subscriber.py
├── package.xml
└── setup.py
```

### constants.py

Centralised names for all topics and services. Import instead of hardcoding strings:

```python
from learning.constants import TOPIC_STATUS, TOPIC_COMMAND
```

| Constant | Value |
|---|---|
| `TOPIC_STATUS` | `/learning/status` |
| `TOPIC_COMMAND` | `/learning/command` |

### Topics

#### /learning/status

Publishes robot status once per second.

**Type:** `std_msgs/msg/String`
**Format:** `battery=<value>; mode=<value>; speed=<value>`

| Field | Behaviour |
|---|---|
| `battery` | Decreases by 0.5–1.5 per second, resets to 100 at 0 |
| `mode` | Alternates between `manual` and `auto` |
| `speed` | Random value 0.0–1.0 |

```bash
# Run
ros2 run learning topics_status_pub
ros2 run learning topics_status_sub

# Inspect
ros2 topic echo /learning/status
ros2 topic info /learning/status
```

---

#### /learning/command

Publishes a robot command every 2 seconds, cycling through the full sequence.

**Type:** `std_msgs/msg/String`
**Commands:** `forward` → `backward` → `left` → `right` → `stop` → repeat

```bash
# Run
ros2 run learning topics_command_pub
ros2 run learning topics_command_sub

# Inspect
ros2 topic echo /learning/command

# Send manually
ros2 topic pub /learning/command std_msgs/msg/String "{data: 'forward'}"
ros2 topic pub --once /learning/command std_msgs/msg/String "{data: 'forward'}"
```
